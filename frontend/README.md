# MEW2 Frontend

## Authentication & Admin Setup

### First-Time Admin Signup
The application requires an initial administrator to be created.
1. Set the `VITE_ADMIN_SECRET` environment variable (e.g., in `.env` or Vercel/Netlify settings) to a secure string.
2. Sign up on the app using this secret as the "Invite Code".
3. The account will be created with `isAdmin: true`.

### Disabling Admin Secret
**Important:** After creating your first admin account, you should disable the secret creation path to prevent unauthorized admin access.
1. Remove `VITE_ADMIN_SECRET` from your environment variables.
2. Redeploy the application.
3. New users will now require an invite code generated by an existing admin.

## Development & Testing

```bash
npm install              # Install dependencies
npm run dev              # Start Vite development server on http://localhost:5173
npm test                 # Run Vitest unit & component test suite (`vitest run`)
npm run test:watch       # Run Vitest in interactive watch mode (`vitest`)
npm run test:coverage    # Generate code coverage report (`vitest run --coverage`)
npm run build            # Run TypeScript type check (`tsc -b`) and production build
```

### Test-Driven Development (TDD)
All contributions to this codebase must follow **Test-Driven Development (TDD)**:
1. Write failing unit (`*.test.ts`) or component (`*.test.tsx`) tests first capturing the expected behavior or bug.
2. Write the minimal code needed to pass the tests (`npm test`).
3. Refactor while keeping all tests green (`npm test` & `npm run build`).
